System Dependency Map

RootTrace AI uses a system dependency map to show how each part of the simulated e-commerce checkout system is connected.

Frontend Checkout Page

Depends on Payment API. Allows users to click checkout and complete payment.

Payment API

Depends on Database. Processes payment requests and retrieves order data.

Promo Code Service

Depends on Database. Checks discount codes and coupon records.

Database

Stores coupon, order, and payment information.

How RootTrace Traces Back

Frontend Error → Payment API Error → Database / Gateway Issue → Likely Root Cause

Instead of treating every alert as a separate problem, RootTrace checks which component depends on another component and traces backwards.

Incident Scenarios

These scenarios are used to test whether RootTrace AI can explain the correct root cause and causal chain.

Scenario 1

Promo Code Bug

A promo code update causes database issues, which later affects payment and checkout.

Root Cause:
Git Commit #402 caused an issue in promo code validation logic.

Git Commit #402 → Promo Code Service Bug → Database Issue → Payment API Timeout → Checkout Failure

Scenario 2

Payment Gateway Timeout

The checkout page shows payment failure, but the issue starts from the external gateway.

Root Cause:
External payment gateway latency increased.

Gateway Issue → Payment API Timeout → Frontend Payment Failure → User Cannot Checkout

Scenario 3

Database Overload

The checkout page freezes because the backend database is overloaded during peak traffic.

Root Cause:
Database overload prevented proper checkout requests.

Database Overload → Payment API Cannot Retrieve Records → Checkout Freezes
